Project Overview & Goals
* Building comprehensive medication tracking app for DART animal rescue
* Primary purpose: track medication administration by foster volunteers for Department of Agriculture compliance
* Secondary goal: create scalable SaaS product for other rescues (pro bono development for DART, then commercialize)
* DOA compliance gap: missing medication administration records by volunteers
* Current deficiency identified during DOA inspections as area needing improvement
* Cloud-based solution for real-time access from anywhere (€5-6/month storage cost estimate)
Current System Limitations & Pain Points
* Excel-based tracking system extremely manual and time-intensive for Francisco
* Volunteers reluctant to input data into spreadsheets (TNR experiment failed - only 2 of many volunteers participated)
* Missing medication administration records creating DOA compliance risk
* Historical data loss incidents due to volunteer departures (cat data lost in 2023 when volunteer left with laptop)
* No centralized real-time access - Francisco chasing people for updates months later
* Invoice reconciliation difficult without proper animal naming conventions
* Foster communication gaps (example: kitten hospitalized Monday, Francisco unaware until vet called)
App Architecture & User Access Levels
* Back office web app (admin level)
o Full database access for Francisco and designated admins
o Comprehensive animal register with all historical data
o Foster management and pet assignment workflows
o Report generation for DOA compliance
o Invoice reconciliation capabilities
* Mobile admin version for field work
o TNR data capture and management
o New animal intake forms with location services
o Vet appointment booking and management
o Real-time data entry during rescue operations
* Foster mobile app (restricted access)
o Access only to animals currently in their care
o Medication logging with dropdown menus and free text
o Treatment history viewing for assigned pets
o Vet appointment requests vs direct booking (permission-based)
o Pet profile display with vet reference numbers, microchip details
Animal Data Structure & Naming Convention
* Naming convention critical for DOA compliance and invoice tracking
o Required format: Location + Source + Description + Volunteer initials
o Examples: “Brabison Hall Beddystown mother Kitten1234”, “Telebook Court Slain Road Surrender Kitten1&2”
o Prevents meaningless names like “Snoopy” or “Kitten” on invoices
* Separate nickname field for foster convenience while maintaining official naming
* Multi-species support with species dropdown: cats (primary), dogs, rabbits, ferrets
* Comprehensive data fields:
o Breed (dropdown): Domestic Short Hair, Domestic Long Hair, British Short Hair, etc.
o Color, gender, neuter status (intact/neutered)
o Age at intake (critical for calculating birth date)
o Microchip number and implant date
o Vaccination records with dates and types
o FIV/FELV test results
o Location tracking: foster care, vet facility, center
o Vet reference numbers (All Pets has system, Westgate uses names only)
o DART volunteer responsible for oversight
Medication Tracking & Treatment Protocols
* DOA compliance requirements: animal name, medication type, dosage, frequency, administrator identity, disposal method
* Open-ended medication system with common defaults:
o Standard treatments: deworming (Milpro, Milbemax), flea treatment (NexGard Combo, Bravecto Plus)
o Antibiotics (various types), steroids (typically Prednisone), anti-inflammatories (Moxicone)
o Cat flu treatments: antibiotics, eye drops/gels (some with steroids)
o Special cases: inhalers, suspensions requiring mixing instructions
* Protocol automation potential for standard treatments:
o Deworming protocol: kittens every 2 weeks from 2-3 weeks old with Locuperazol
o Transition to solid wormers (Milpro/Milbemax) monthly until 6 months
o Automated reminders for multi-day treatments
* Weight recording at vet visits for accurate dosage calculations
* Educational popups planned (Phase 2): medication explanations, vaccine information, treatment protocols
Foster Management & Permissions System
* Current volunteer base: ~50 total volunteers, subset are active fosters
* Foster capacity: up to 12 animals per experienced foster (some also handle dogs)
* Two-tier vet appointment authorization system:
o Experienced fosters: checkbox permission for direct vet booking
o New/untrusted fosters: request-based system requiring admin approval
o Authorized list maintained per vet clinic for continuity
* Foster onboarding workflow:
o Digital foster agreement form (replaces current paper system)
o Equipment tracking: beds, litter boxes, carriers must be returned
o Pet assignment system with real-time updates
o Historical tracking of all animals fostered by each volunteer
* Foster agreement covers: care responsibilities, equipment return, contact protocols, DART representation guidelines
TNR Module & Workflow Integration
* Separate TNR tracking system (currently 280+ cats in 2024, can reach 300+)
* TNR-specific data capture: location, contact person, volunteer assignments, completion status
* Integration pathway: TNR cats marked for rehoming transfer to main foster system
* Current challenge: volunteers not inputting TNR data (only Rafael and Francisco actively updating)
* Mobile TNR interface for field volunteers to improve data capture compliance
Reporting & DOA Compliance Framework
* Annual DOA grant application process (typically May submission):
o Part A: organizational evidence and capacity
o Part B: detailed animal throughput data, financials, rehoming figures
o Final interview: detailed record review with drill-down capability
* Required reporting capabilities:
o Animal throughput statistics by year
o Medication management documentation
o TNR activities (separate reporting requirement)
o Financial integration for invoice reconciliation
* Export formats: CSV for data analysis, PDF for formal submissions
* Dashboard for board presentations: current animal counts, adoption rates, financial summaries
* Filter capabilities: by date ranges, treatment types, outcomes (adopted, PTS, reclaimed, TNR returned)
Data Migration & Historical Records
* Historical data scope varies by species:
o Cats: 2022 onwards (4 years) due to 2023 data loss incident
o Dogs: longer historical record available
o TNR: records since 2004 but needs cleanup and consolidation
* Migration approach: import existing Excel data as-is, add legacy notes field for historical context
* New animals from July 2026: full structured data capture with detailed treatment histories
* GDPR compliance: volunteer data anonymization process, right to deletion, data export capabilities
Development Phases & Feature Roadmap
* MVP (Phase 1): Core medication tracking, basic foster app, admin reporting
* Phase 2 enhancements:
o Educational popups with treatment protocols
o Automated protocol reminders
o Photo storage for pet identification
o Advanced dashboard analytics
o Volunteer training record integration
* Scalability consideration: design for multi-rescue deployment after DART beta testing
Action Items & Next Steps
* Francisco to provide detailed field requirements questionnaire with data types and validation rules
* Lisa to share DOA documentation requirements and previous year’s submissions for analysis
* Separate development track for TNR module with integration points defined
* GDPR compliance framework implementation from launch
* Cost analysis and monthly fee structure for cloud hosting
* MVP scope definition focusing on core medication tracking before adding advanced features
* Beta testing timeline establishment with DART volunteers
* Volunteer onboarding strategy for app adoption
