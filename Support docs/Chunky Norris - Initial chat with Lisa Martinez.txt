Current Data Management System & Challenges
Data Fields Currently Tracked
* Animal intake information:
o Intake date and departure date
o Source classification (stray, surrender, TNR)
o Gender, age, coat type and color
o Foster placement details and location
* Medical tracking:
o Vaccination dates and neutering dates
o FIV/FELV testing results (cats only)
o Basic medication notes (incomplete)
* Outcome documentation:
o Rehomed, reclaimed, euthanized, or died in care
o Adopter handoff information including medical history
Process Gaps & Volunteer Adoption Issues
* Excel-based system creates bottlenecks
o Only Lisa and Rafael consistently use shared TNR spreadsheet
o 2 out of 4 volunteers actively input data
o Other volunteers avoid technology due to unfamiliarity
* Foster medication tracking failures:
o Fosters should write treatments in notebooks but often forget
o “I’ll do it later” mentality leads to lost information
o No standardized process for capturing foster-administered medications
* Data input resistance factors:
o Limited patience with technology problems
o Unwillingness to troubleshoot application issues
o Time constraints and competing priorities
Compliance Risks
* Department of Agriculture annual grant review identified medication control as primary weakness
* Missing documentation for animals in foster care
* Inadequate home check records for dog adoptions
* Separate TNR tracking required but underutilized
Department of Agriculture Requirements
Mandatory Documentation Standards
* Medication administration tracking must include:
o Specific medication administered (e.g., deworming medication name)
o Dosage and administration date
o Person who administered treatment
o Animal weight at time of treatment
* Foster care oversight:
o Complete medication logs for all treatments (cat flu, dewormings, flea treatments)
o Documentation must be available for adopter handoff
* Home check documentation:
o Physical property inspection results
o 35-question assessment responses
o Garden/outdoor space evaluation for dogs
Reporting & Export Needs
* Annual statistics compilation required
* CSV export capability essential for data manipulation
* Specific instance documentation for audit purposes
* Mortality tracking (euthanized vs. natural death)
Proposed Application Architecture
Dual Interface Design
* Admin dashboard for Francisco:
o Excel-like bulk data entry capabilities
o Complete animal record management
o Reporting and export functions
* Mobile foster interface:
o Simplified medication logging
o Species-specific field loading
o Minimal steps for quick entry
Proposed Features
* Dynamic form generation based on animal species
* Photo attachment capabilities for medical records
* Phone-based data entry during conversations
* Shared drive integration for volunteer access
* Home check digitization with mobile forms
* Automated field population (e.g., FIV/FELV for cats only)
User Roles & Journeys
Admin User (Francisco)
1. Bulk data entry from volunteer notes/calls
2. Generate compliance reports for Department of Agriculture
3. Export data to CSV for grant applications
4. Manage foster assignments and animal placements
5. Review and validate volunteer-entered data
Foster/Volunteer User
1. Receive animal assignment with mobile app link
2. Log medications using simple form (pet name + treatment details)
3. Upload photos of treatments or medical notes
4. Submit data with minimal technical barriers
5. Access animal history for adopter handoff
Home Check Inspector
1. Access digital questionnaire on mobile device
2. Complete 35-question assessment during visit
3. Document garden/property observations
4. Submit results for Department of Agriculture compliance
Requirements Backlog
P1 (Critical - Department of Agriculture Compliance)
* Medication tracking system
o Acceptance criteria: Capture medication, dosage, date, administrator
o Risk: Grant funding dependent on compliance
* Foster mobile interface
o Acceptance criteria: <3 steps to log treatment
o Risk: Volunteer adoption failure without simplicity
* CSV export functionality
o Acceptance criteria: All data exportable for reporting
o Risk: Manual reporting creates compliance gaps
P2 (High - Operational Efficiency)
* Home check digitization
o Acceptance criteria: Replace 35-question email process
o Risk: Current process adequate but inefficient
* Species-specific field loading
o Acceptance criteria: Automatic FIV/FELV fields for cats
o Risk: User confusion without proper field management
* Photo attachment capabilities
o Acceptance criteria: Support medical documentation photos
o Risk: Storage costs and data management complexity
P3 (Medium - Enhanced Features)
* Bulk data entry from phone conversations
o Acceptance criteria: Admin can input during calls
o Risk: Feature complexity may delay core functionality
* TNR workflow separation
o Acceptance criteria: Dedicated feral cat tracking
o Risk: Workflow duplication if not properly integrated
Metrics & Success Criteria
Adoption Metrics
* Volunteer data entry participation rate >75%
* Foster medication logging compliance >90%
* Time reduction in Francisco’s data entry workload >50%
Compliance Metrics
* 100% medication tracking for animals in care
* Complete home check documentation for all dog adoptions
* Successful Department of Agriculture audit with no data gaps

* Data migration strategy from current Excel spreadsheets
Process Dependencies
* Volunteer training requirements for new system
* Department of Agriculture documentation review needed
* Monday.com evaluation as potential alternative solution
Outstanding Questions
* Specific medication fields required beyond basic tracking
* Home check questionnaire digitization priority vs. custom development
* Data retention requirements and backup strategies
* Multi-rescue solution scalability for future commercial offering
